JUnit 5 tests do not need public visibility.


The public and protected modifiers can be useful when tests need to be overrideable, however this is the exception. In general, tests should be package-private to prevent any interference like naming collisions or accidental overrides.


Before 
@Test
public void shouldReturn200OkForValidPath() throws Exception {
    // Perform the test
}

After 
@Test
void shouldReturn200OkForValidPath() throws Exception {
    // Perform the test
}
