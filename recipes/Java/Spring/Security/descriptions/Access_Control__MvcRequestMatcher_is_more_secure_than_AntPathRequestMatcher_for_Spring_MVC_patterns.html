For endpoint authorization, it is recommended to use mvcMatchers instead of antMatchers, as it will provide authorization for all paths that the @RequestMapping annotation understands. In other words, antMatchers will only authorise /admin, and leave /admin/, which leads to the same endpoint, accessible for unauthorised users. mvcMatchers will protect both paths.


Before 
http.authorizeRequests().antMatchers(“/admin”).hasRole("ADMIN");

After 
http.authorizeRequests().mvcMatchers(“/admin”).hasRole("ADMIN");
