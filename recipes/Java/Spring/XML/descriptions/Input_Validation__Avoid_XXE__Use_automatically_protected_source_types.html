Secure coding practices prescribe to pass source implementations that do not require manual parsing to the unmarshaller, i.e. either SAXSource or StreamSource.


There are four possible source implementations passed to the unmarshaller: DOMSource, StAXSource, SAXSource and StreamSource.

  • For a DOMSource, the XML has already been parsed by user code and that code is responsible for protecting against XXE.
  • For a StAXSource, the XMLStreamReader has already been created by user code and that code is responsible for protecting against XXE.
  • For SAXSource and StreamSource instances, Spring has disabled external entity processing by default, protecting it from XXE.

    • It is recommended to use source implementations where external entity processing is handled by the framework, i.e. SAXSource or StreamSource.
