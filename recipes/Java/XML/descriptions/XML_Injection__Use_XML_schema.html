The range of allowed XML documents and features should be limited to prevent injection attacks and other logic issues.


Using a strict XML Schema enables developers to describe and enforce which elements are allowed in the XML documents processed by the application. The schema can limit the number of occurrences of elements to prevent overflows or can explicitly indicate the structure of the document to prevent type mismatches.


factory.setSchema(Schema);

Where factory is an instance of:

  • DocumentBuilderFactory
  • TransformerFactory
  • SAXParserFactory
  • SchemaFactory
  • XPathFactory
