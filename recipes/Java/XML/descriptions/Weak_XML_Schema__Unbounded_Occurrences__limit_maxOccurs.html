Sequences in XML documents should be limited to prevent logic issues such as overflows.


Using a strict XML Schema enables developers to describe and enforce which elements are allowed in the XML documents processed by the application. The schema should limit the number of occurrences of elements to prevent overflows.


Example code: 
<element>
  <complexType>
    <sequence minOccurs="1" maxOccurs="unbounded">
      <element type="string"></element>
    </sequence>
  </complexType>
</element>
