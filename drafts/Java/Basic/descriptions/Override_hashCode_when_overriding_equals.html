Classes that override the equals method should also override the hashCode method


The java.lang.Object class of the Java SE documentation notes: "It is generally necessary to override the hashCode method whenever this method is overridden, so as to maintain the general contract for the hashCode method, which states that equal objects must have equal hash codes."


Before 
public class MyClass {
@Override
public boolean equals(Object o) {
    ...
}

After 
public class MyClass {
@Override
public boolean equals(Object o) {
    ...
}

@Override
public int hashCode() {
    return Objects.hash(/* list properties here */);
}
