Not having a custom error page for javax.servlet.ServletException could lead to information leakage.


Default error pages often contain full stack traces of the errors that are encountered. This discloses information about the inner workings of the application that could be abused by attackers.


Before 
<web-app>
...
</web-app>
After 
<web-app>
    <error-page>
        <error-code>javax.servlet.ServletException;/error-code>
    </error-page>
</web-app>
